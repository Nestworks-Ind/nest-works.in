What it does

Everything you need to run and analyse JMeter tests.

One desktop app replaces the JMeter command line, Grafana dashboards, and manual log analysis — with a clean UI that shows you exactly what is happening.

NestPulse is a native Windows desktop application — installed on your machine, runs completely offline. No subscription. No cloud. No data ever leaves your machine.

Every step a performance engineer does manually — handled end-to-end from one window.

01
One-click test execution
Select a client and JMX script, configure thread groups, and start the test — no terminal, no JMeter GUI required. Runs JMeter as a background process.
02
Live aggregate monitor
Real-time table updates every 2 seconds — active users, throughput, avg response time, 90th percentile, and error rate per transaction. Matches JMeter's Aggregate Report exactly.
03
Error inspector
Opens in a separate window. Shows every failed request with full request headers, body, response headers, and response data. Available live during a run and after completion.
04
Scheduled test runs
Schedule tests to run once, daily, or weekly at a specific time — no cron, no scripts. Uses the exact client, JMX, and thread group configuration from the Execution tab.
05
Excel export
Export aggregate results to Excel in one click. Contains all transaction data with samples, averages, percentiles, throughput, and error rates.
06
File management
Upload, edit, and delete JMX scripts and CSV test data files directly through the app. Organised per client — no file explorer needed.
07
Run history & HTML reports
Every run is stored in the client folder with JTL, log, and used JMX. Generate JMeter HTML reports on demand directly from the Results tab.
08
Thread group override
Edit users, ramp-up, duration, and delay for each thread group before every run — without modifying the JMX file. Enable or disable individual thread groups per run.

How it works

From JMX script to
live results in seconds.

Seven screens. One desktop window. No command line.

01
Launch & Select a Client

Open NestPulse and you land on the Execution tab. Select your client from the dropdown — each client has its own JMX scripts, test data, and results folder. No setup required after first-time configuration.

Client dropdown JMX file selector Load scenarios
02
Configure & Start the Test

After loading scenarios, all thread groups from the JMX are shown in a table. Edit users, ramp-up, duration, and delay per group — or leave them as-is. Hit Start test and NestPulse launches JMeter immediately.

Thread group editor Enable / disable groups Start test
03
Schedule Tests to Run Automatically

Click Schedule this test on the Execution tab to set up automated runs. Choose run-once, daily, or weekly — NestPulse handles execution at the right time using the exact same client and JMX configuration.

Run once / Daily / Weekly Date & time picker Active schedules list
04
Watch Live Metrics Update in Real Time

As the test runs, the Live Monitor tab shows active users, throughput, avg response time, and error rate — refreshed every 2 seconds. The aggregate table below shows per-transaction metrics that match JMeter's Aggregate Report exactly.

6 metric cards Aggregate table 2 sec refresh Exact P90 / P95 / P99
05
Inspect Every Error in Detail

Click Error Inspector to open a dedicated window showing all failed requests. Each error shows the full request headers, request body, response headers, and response data — available live during the run and after it completes.

Request headers & body Response headers & data Live + post-run Search & filter
06
Browse Results & Export to Excel

The Results tab shows every run with status, timestamps, and quick-access downloads. Expand any row to get the JTL, log, and used JMX. Export aggregate results to Excel or generate the full JMeter HTML report in one click.

Run history Excel export HTML report JTL & log download
07
Manage JMX Scripts & Test Data

The Manage Files tab lets you upload, edit, and delete JMX scripts and CSV test data files per client — all through the UI. No file explorer, no path editing. NestPulse automatically wires data file paths into the JMX at run time.

Upload JMX & CSV In-app CSV editor Per-client organisation
